# Runbook

## Local Health Check

```bash
curl http://localhost:8080/healthz
curl http://localhost:8080/readyz
curl http://localhost:8080/metrics
```

## Common Issues

### Port already in use

Change the port mapping in `docker-compose.yaml` or stop the process using port 8080.

### Container starts but pages do not load

Check that the `static/` directory exists inside the image:

```bash
docker run --rm -it --entrypoint /app/go-web-app go-web-app-devops:local
```

### Helm deployment fails

Validate templates first:

```bash
helm lint helm/go-web-app-chart
helm template go-web-app helm/go-web-app-chart
```

### Trivy scan fails CI

Review the vulnerability output, update base images or dependencies, and only suppress findings when there is a documented reason.

## Rollback Strategy

Use immutable image tags generated by GitHub Actions. Roll back by updating Helm values to the previous working image tag and redeploying.
